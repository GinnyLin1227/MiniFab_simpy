# MiniFab Integrated Training System - Complete Guide

## 🎯 Project Overview

A **production-ready distributed reinforcement learning system** for manufacturing job scheduling using:

1. **Ray Distributed Training** - Parallel environment evaluation (4x speedup)
2. **Event-Driven Decision System** - Automatic 12D feature extraction at decision points  
3. **Evolution Strategies (ES)** - Peer comparison evolution (no ground truth needed)

## 🎓 Understanding the System

### 1. Event-Driven Decision (決策點事件驅動)

**Flow:**

```
Machine completes processing
    ↓
Extract 12 Features (automatic)
    ├─ Machine state (4): utilization, setup_count, pending_lots, idle_time
    ├─ Queue state (4): length, max_age, product_variety, downstream_util
    └─ Lot state (4): remaining_steps, wait_time, product_type, batch_potential
    ↓
Neural Network Decision (highest score lot)
    ↓
Schedule Future Event (register completion in event queue)
    ↓
Continue simulation
```

### 2. Evolution Strategies (演化策略)

**Key Insight:** No need for "ground truth" optimal solution

```
Traditional:           ES Peer Comparison:
Strategy vs Optimal   Strategy vs Strategy
❌ No optimal exists   ✅ Relative evaluation
                      ✅ Self-adaptive evolution
                      ✅ Population diversity maintained

Fitness = Win Rate + Mean Reward
```

**Process:**

```
Gen t: Evaluate population (20 strategies vs each other)
    ↓
  Select Elite (top 4 by fitness)
    ↓
  Evolve (mutate elite → offspring)
    ↓
Gen t+1: Repeat...

Adaptive Mutation:
- Good individuals: lower mutation (conservative)
- Poor individuals: higher mutation (exploratory)
```

### 3. Ray Parallel Training (Ray分散式訓練)

**Architecture:**

```
20 strategies × 3 episodes = 60 env steps

Sequential: ════════════════════════════════ (60 time units)
Parallel:   ════════ (15 time units) ← 4x speedup!

Actor 1: Strategies 1-5 × 3 episodes
Actor 2: Strategies 6-10 × 3 episodes
Actor 3: Strategies 11-15 × 3 episodes
Actor 4: Strategies 16-20 × 3 episodes
```

## 💻 Configuration Guide

### Basic Configuration (Default)

```bash
python integrated_training.py
# Uses: 10 generations, 3 episodes, population=20, 4 actors
```

### For Different Hardware

```bash
# Laptop (4 cores)
python integrated_training.py --ray-actors 2 --population-size 10

# Workstation (8 cores)
python integrated_training.py --ray-actors 4 --population-size 20

# High-performance (16+ cores)
python integrated_training.py --ray-actors 8 --population-size 40
```

### For Different Training Styles

```bash
# Fast convergence (risk: local optima)
python integrated_training.py \
    --generations 20 \
    --population-size 30 \
    --elite-size 8

# Conservative exploration (slow but better quality)
python integrated_training.py \
    --generations 5 \
    --episodes-per-gen 5 \
    --population-size 50

# Quick testing
python integrated_training.py --generations 2 --episodes-per-gen 1
```

### Full Parameter Reference

```bash
python integrated_training.py --help

# Output:
# --generations         Number of ES generations (default: 10)
# --episodes-per-gen    Episodes per policy per generation (default: 3)
# --population-size     ES population size (default: 20)
# --elite-size          Number of elite policies (default: 4)
# --ray-actors          Number of parallel Ray actors (default: 4)
# --output-dir          Results directory (default: ./training_results)
```

### Parse Results (Python)

```python
import json

with open("./training_results/training_report.json") as f:
    report = json.load(f)

# Best fitness trend
fitnesses = [g["best_fitness"] for g in report["training_history"]]
print(f"Improvement: {fitnesses[-1] - fitnesses[0]:.1f}")

# Best policy
best = report["best_individual"]
print(f"Best ID: {best['individual_id']}, Fitness: {best['fitness']}")
```

## 🐛 Troubleshooting

### Issue: Ray initialization fails

**Solution:**
```bash
pip install --upgrade ray
python -c "import ray; ray.init(); print('OK')"
```

### Issue: Training is slow

**Solutions:**
- Increase `--ray-actors` (if hardware allows)
- Reduce `--episodes-per-gen` (faster evaluation)
- Reduce `--generations` (less total iterations)

### Issue: Out of memory

**Solutions:**
- Reduce `--ray-actors` (4 → 2)
- Reduce `--population-size` (20 → 10)
- Use smaller `--episodes-per-gen`

### Issue: Can't find minifab modules

**Solution:**
```bash
# Make sure you're in the right directory
cd muti-agent\ split\ python/

# Or add to Python path
export PYTHONPATH="${PYTHONPATH}:$(pwd)"
```

See full troubleshooting in `TRAINING_GUIDE.md`
